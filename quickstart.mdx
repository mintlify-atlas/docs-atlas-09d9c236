---
title: "Quickstart"
description: "Get ElectroHome running on your local machine in under 10 minutes."
---

<Steps>
  <Step title="Check prerequisites">
    Make sure you have the following installed before continuing:

    - **Python 3.10 or higher** — check with `python --version`
    - **pip** — check with `pip --version`
    - **Git** — check with `git --version`

    <Tip>
      On macOS, use [Homebrew](https://brew.sh) to install Python: `brew install python@3.12`. On Windows, download the installer from [python.org](https://www.python.org/downloads/).
    </Tip>
  </Step>

  <Step title="Clone the repository">
    ```bash
    git clone https://github.com/your-org/electrohome.git
    cd electrohome
    ```

    <Note>
      Replace the URL above with the actual repository URL if it differs.
    </Note>
  </Step>

  <Step title="Create and activate a virtual environment">
    Create an isolated Python environment for the project:

    ```bash
    python -m venv .venv
    ```

    Then activate it:

    <Tabs>
      <Tab title="macOS / Linux">
        ```bash
        source .venv/bin/activate
        ```
      </Tab>
      <Tab title="Windows (PowerShell)">
        ```powershell
        .venv\Scripts\Activate.ps1
        ```
      </Tab>
      <Tab title="Windows (Command Prompt)">
        ```cmd
        .venv\Scripts\activate.bat
        ```
      </Tab>
    </Tabs>

    Your prompt should now show `(.venv)` to confirm the environment is active.
  </Step>

  <Step title="Install dependencies">
    ```bash
    pip install -r requirements.txt
    ```

    This installs Django 5.2 and all other packages required by ElectroHome.
  </Step>

  <Step title="Set environment variables">
    Create a `.env` file in the project root with the following variables:

    ```bash
    SECRET_KEY=your-secret-key-here
    DEBUG=True
    DATABASE_URL=sqlite:///db.sqlite3
    ```

    | Variable | Description |
    |---|---|
    | `SECRET_KEY` | Django secret key used for cryptographic signing. |
    | `DEBUG` | Set to `True` for local development. **Never set this to `True` in production.** |
    | `DATABASE_URL` | Database connection string. SQLite is the simplest option for local development. |

    <Warning>
      Never commit your `.env` file to version control. Add it to `.gitignore` if it is not already there.
    </Warning>

    <Tip>
      Generate a strong `SECRET_KEY` with Python:

      ```bash
      python -c "from django.core.management.utils import get_random_secret_key; print(get_random_secret_key())"
      ```
    </Tip>
  </Step>

  <Step title="Apply database migrations">
    Run Django's migration command to create the database schema:

    ```bash
    python manage.py migrate
    ```

    Django will apply all pending migrations and print each one as it runs. You should see output ending with:

    ```
    Applying ...... OK
    ```
  </Step>

  <Step title="Create a superuser">
    Create an admin account to access the Django admin panel:

    ```bash
    python manage.py createsuperuser
    ```

    You will be prompted for a username, email address, and password.

    <Note>
      The password must meet Django's default strength requirements. Use at least 8 characters with a mix of letters and numbers.
    </Note>
  </Step>

  <Step title="Start the development server">
    ```bash
    python manage.py runserver
    ```

    Django starts a local server with output similar to:

    ```
    Watching for file changes with StatReloader
    Performing system checks...

    System check identified no issues (0 silenced).
    Django version 5.2, using settings 'electrohome.settings'
    Starting development server at http://127.0.0.1:8000/
    Quit the server with CONTROL-C.
    ```
  </Step>

  <Step title="Open the admin panel">
    Visit [http://localhost:8000/admin/](http://localhost:8000/admin/) in your browser and log in with the superuser credentials you created.

    From the admin panel you can manage products, orders, users, and all other ElectroHome data.

    <Check>
      If you can log in to the admin panel, your local setup is complete.
    </Check>
  </Step>
</Steps>
